# 幕启 MuQi 功能测试脚本

## 测试说明

由于未安装Playwright，我们将使用以下替代方案进行测试：

### 方案1: 手动测试（推荐）

1. **打开浏览器访问**: http://localhost:3001/

2. **测试步骤**:
   - [ ] 页面正常加载
   - [ ] 标题显示："粘贴你的小说，生成结构化剧本"
   - [ ] 左侧有文本输入区
   - [ ] 右侧有输出区域
   - [ ] 有"开始转换"按钮

3. **功能测试**:
   - [ ] 输入示例小说内容
   - [ ] 点击"开始转换"
   - [ ] 等待转换完成
   - [ ] 查看YAML输出
   - [ ] 检查输出格式（包含meta, acts, characters, dialogues）
   - [ ] 切换到"预览"标签
   - [ ] 测试"清空"按钮
   - [ ] 测试"复制"按钮
   - [ ] 测试"下载"按钮

### 方案2: 使用构建版本测试

1. **打开文件**: `dist/index.html`
2. **执行相同的手动测试步骤**

### 方案3: API测试

使用curl测试服务器是否正常运行：

```bash
curl http://localhost:3001/
```

如果返回HTML内容，说明服务器正常。

2. **test_full_novel.txt** - 6个章节的完整测试（推荐）

## 预期结果

### YAML输出应包含:
- `script:` - 根节点
- `meta:` - 元数据（title, source, format, generated_at, stats）
- `acts:` - 幕列表
  - `act_id` - 幕编号
  - `act_title` - 幕标题（应为章节标题）
  - `scenes:` - 场景列表
    - `scene_id` - 场景编号
    - `location` - 场景位置
    - `time` - 时间
    - `mood` - 氛围
    - `scene_note` - 旁白
    - `characters:` - 角色列表
      - `name` - 角色名
      - `role` - 角色类型
    - `dialogues:` - 对话列表
      - `dialogue_id` - 对话编号
      - `type` - 对话类型（spoken/inner/offscreen）
      - `speaker` - 说话人
      - `line` - 台词
      - `action` - 动作
      - `note` - 备注

## 验收标准

- ✓ 页面正常加载

- ✓ 能触发转换功能
- ✓ YAML输出格式正确
- ✓ 章节标题正确显示在act_title中
- ✓ 角色名称被正确提取
- ✓ 对话被正确提取
- ✓ 预览功能正常
- ✓ 复制和下载功能可用
